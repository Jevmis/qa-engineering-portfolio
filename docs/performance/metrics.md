# Performance Metrics

> Measuring application performance using meaningful engineering metrics.

---

# Overview

Performance metrics provide objective insight into how an application behaves under different workloads.

Rather than relying on assumptions, engineering teams can use measurable data to identify bottlenecks, compare releases, and make informed optimisation decisions.

---

# Key Metrics

## Response Time

The time taken for the application to respond to a request.

Lower response times generally improve user experience.

---

## Throughput

The number of requests or transactions processed within a given period.

Higher throughput usually indicates better system capacity.

---

## Error Rate

The percentage of failed requests during testing.

Monitoring error rates helps identify system instability under load.

---

## Concurrent Users

The number of users actively interacting with the application at the same time.

---

## Transactions Per Second (TPS)

Measures how many business transactions the application completes each second.

---

## CPU Utilisation

Tracks processor usage during testing.

High CPU utilisation may indicate performance bottlenecks.

---

## Memory Usage

Monitors memory consumption throughout test execution.

Unexpected increases may indicate memory leaks or inefficient resource management.

---

## Network Performance

Evaluates data transfer efficiency between clients and servers.

---

# Why Performance Metrics Matter

Monitoring these metrics helps teams:

- Identify bottlenecks
- Compare application versions
- Validate optimisation efforts
- Support capacity planning
- Improve scalability
- Increase release confidence

---

# Continuous Improvement

Performance testing is most valuable when metrics are monitored consistently across releases, allowing engineering teams to identify trends and continuously improve system performance.
