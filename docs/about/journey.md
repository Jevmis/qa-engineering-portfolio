# Career Journey

> Every engineer has a story. Mine is one of curiosity, continuous learning, leadership, and a passion for delivering quality software.

---

# Where It All Began

My journey into technology started with a strong interest in solving problems and understanding how software works behind the scenes.

Studying Computer Engineering provided me with a solid technical foundation, but I quickly discovered that building software was only part of the equation. Delivering reliable, high-quality software required disciplined testing, structured processes, and a commitment to continuous improvement.

That realization led me into the world of Software Quality Assurance.

---

# Discovering Quality Engineering

I began my career focusing on manual software testing, learning the importance of understanding business requirements, identifying defects, and validating user expectations.

This stage helped me develop essential skills in:

- Functional Testing
- Regression Testing
- User Acceptance Testing (UAT)
- Test Case Design
- Bug Reporting
- Requirements Analysis
- Software Documentation

Working closely with developers, business analysts, and project managers taught me that quality is a shared responsibility across the entire software development lifecycle.

---

# Growing Beyond Manual Testing

As projects became larger and more complex, I recognized the need for automation to improve testing efficiency and reliability.

This led me to invest significant time in learning:

- JavaScript
- TypeScript
- Cypress
- Playwright
- API Testing
- SQL Validation

I started building reusable automation frameworks that reduced repetitive manual work and improved regression testing coverage.

Automation became more than just writing scripts—it became an opportunity to improve engineering workflows and help teams deliver software faster with greater confidence.

---

# Taking on Leadership

As my experience grew, I transitioned into leadership roles where my responsibilities expanded beyond testing.

I began leading QA activities by:

- Planning testing activities
- Defining QA processes
- Mentoring junior QA engineers
- Reviewing bug reports
- Coordinating regression testing
- Improving documentation
- Supporting release planning
- Promoting quality best practices

Leading a QA team reinforced the importance of communication, collaboration, and continuous process improvement.

---

# Expanding My Technical Expertise

To become a more effective Quality Engineer, I broadened my knowledge beyond functional testing.

I developed practical experience in:

- REST API Testing
- Performance Testing using Apache JMeter
- SQL Database Validation
- Security Testing using OWASP ZAP
- CI/CD with GitHub Actions
- Technical Documentation
- Automation Framework Architecture

These skills enabled me to contribute throughout the software delivery lifecycle rather than focusing solely on testing.

---

# Building Better QA Processes

One area I particularly enjoy is improving the way teams work.

Throughout my career, I have created and maintained:

- Test Strategies
- Test Plans
- QA Documentation
- Bug Reporting Standards
- Test Templates
- Checklists
- Release Validation Processes
- Knowledge Bases
- QA Workflows

I also developed automated QA dashboards using Google Sheets to help teams track testing progress, defect trends, and quality metrics more efficiently.
