# Security Testing

> Improving software quality by validating security controls, identifying common vulnerabilities, and supporting secure software delivery.

---

# Overview

Security testing helps ensure that software protects sensitive information, enforces proper access controls, and resists common security threats.

While security specialists perform deep penetration testing, Quality Assurance teams play an important role in verifying that applications implement security requirements correctly throughout development.

---

# Areas Covered

My security testing activities include:

- Authentication Testing
- Authorization Verification
- Input Validation
- Session Management
- Secure Configuration Checks
- OWASP Top 10 Awareness
- OWASP ZAP Scanning
- API Security Validation

---

# Objectives

Security testing aims to:

- Reduce application risk
- Protect user data
- Validate access controls
- Detect common vulnerabilities
- Support secure software releases

---

# Tools

Security-related testing may involve:

- OWASP ZAP
- Browser Developer Tools
- Postman
- Swagger / OpenAPI
