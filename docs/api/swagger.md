# Swagger / OpenAPI

> Using API documentation to improve testing accuracy and collaboration.

---

# Overview

Well-documented APIs improve communication between developers, testers, and stakeholders.

Swagger (OpenAPI) provides a structured way to understand available endpoints, request formats, response models, and authentication requirements.

---

# Benefits

Swagger helps testers:

- Understand endpoints
- Review request schemas
- Validate response structures
- Identify required parameters
- Explore API capabilities

---

# Typical Usage

During testing, I use Swagger to:

- Review endpoint documentation
- Validate request payloads
- Understand expected responses
- Verify business rules
- Plan automation scenarios

---

# Collaboration

Accurate API documentation benefits the entire engineering team by reducing ambiguity and improving consistency during development and testing.

---

> **"Good documentation makes good testing possible."**
