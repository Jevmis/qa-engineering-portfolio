# Authentication Testing

> Verifying that only authorised users and systems can securely access application resources.

---

# Overview

Authentication testing validates that users and systems can securely access protected resources using approved authentication mechanisms.

It helps ensure that access controls function correctly while preventing unauthorised access.

---

# Areas Covered

Typical authentication testing includes:

- Login Validation
- Logout Functionality
- Session Management
- Token Validation
- Access Control
- Role-based Access
- Invalid Credential Handling

---

# Common Authentication Methods

Modern applications commonly use:

- Username & Password
- Bearer Tokens
- API Keys
- OAuth
- JWT (JSON Web Tokens)

---

# Validation Activities

Typical verification includes:

- Successful authentication
- Invalid credentials
- Missing authentication
- Expired tokens
- Invalid tokens
- Permission validation
- Session timeout
- Logout verification

---

# Security Considerations

Authentication testing also verifies that:

- Sensitive information is protected
- Error messages do not expose unnecessary details
- Sessions are properly invalidated after logout
- Access is restricted to authorised users

---

> **"Strong authentication is the first line of defence for every application."**
