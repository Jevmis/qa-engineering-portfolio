# Quality Assurance

> Delivering software quality through structured testing, risk management, effective documentation, and continuous improvement.

---

# Overview

Quality Assurance is more than identifying defects,it's about building confidence in software before it reaches users.

Throughout my career, I have contributed to projects across government agencies, registry platforms, and private-sector organisations, ensuring applications meet functional, business, and quality expectations.

My approach combines structured testing, automation, documentation, collaboration, and continuous improvement.

---

# Core QA Activities

My experience covers the complete software testing lifecycle, including:

- Requirement Analysis
- Test Planning
- Test Strategy
- Functional Testing
- Regression Testing
- Smoke Testing
- User Acceptance Testing (UAT)
- SQL Backend Validation
- API Testing
- Defect Management
- Release Validation

---

# Quality Philosophy

Quality should be considered from the beginning of software development,not only during testing.

By collaborating with developers, business analysts, and stakeholders throughout the development lifecycle, potential issues can be identified earlier, reducing risk and improving delivery confidence.

# Guiding Principles

Every QA initiative I contribute to is guided by a few core principles:

- Understand the business requirements.
- Prioritise risk.
- Test what matters most.
- Communicate clearly.
- Document consistently.
- Continuously improve processes.
